# Environment Variables Configuration

This project uses environment variables to configure API endpoints for different environments.

## Files

- `.env` - Production environment variables (committed to repository)
- `.env.development` - Development environment variables (local only, not committed)
- `.env.local` - Local overrides (not committed)

## Available Variables

### VUE_APP_API_BASE_URL
The base URL for the Note API backend.
- Production: `https://noteapi.lukas-bownik.net`
- Development: `http://localhost:5000` (or your local API URL)

### VUE_APP_ML_BASE_URL
The base URL for the ML service.
- Production: `https://ml.lukas-bownik.net`
- Development: `http://localhost:8080` (or your local ML service URL)

### VUE_APP_ENV
The environment mode.
- Production: `production`
- Development: `development`

## Usage

### Development
Create a `.env.development` file in the project root (already provided) or `.env.local` for personal overrides:

```env
VUE_APP_API_BASE_URL=http://localhost:5000
VUE_APP_ML_BASE_URL=http://localhost:8080
VUE_APP_ENV=development
```

Run development server:
```bash
npm run serve
```

### Production Build
The `.env` file contains production defaults. Build for production:
```bash
npm run build
```

## Notes

- All Vue CLI environment variables must be prefixed with `VUE_APP_`
- Environment variables are embedded at build time
- The production `.env` file is committed to ensure consistent production builds
- Local development configurations (`.env.development`, `.env.local`) are not committed
